Selection

Projects

This page is not just about finished products. Some work is already well underway, some is still taking shape, and some is intentionally kept at the level of a concept or a technical foundation. Taken together, it shows the kinds of problems and structures I tend to work on.

The common thread is usually the same: software and systems should remain understandable, useful and maintainable over time.

Selected work and ongoing ideas

The selection below mixes ongoing work, technical building blocks and projects that are deliberately meant as foundations for future applications.

NimbusJS / NimbusStack

ecosystem ongoing

A JavaScript ecosystem with its own framework core, suite and supporting services. The interesting part is not “yet another framework”, but the attempt to build a coherent technical foundation that can support several projects over time.

What matters here is the architectural side of it: naming, separation of concerns, reuse, and the question of how a technical base can stay clear instead of growing into a pile of unrelated pieces.

SASD\Mustela Logging

PHP building block

A logging framework built around appenders, formatters and configurable targets. It is meant as a reusable base rather than a one-off component for a single project.

Its value lies less in visual appeal and more in structure, extension points and the ability to solve a recurring technical need in a clean way instead of redoing it each time.

REST services for authentication, profiles and projects

PHP service architecture

A set of separate services for authentication, profile management and project logic, each with a clear responsibility and a structure that can be reused beyond a single application.

Work like this rarely looks spectacular on the surface, but it says a lot about how someone thinks: boundaries, responsibilities, security concerns and long-term maintainability.

SASD CMS / web presence

vanilla JS + PHP ongoing

A deliberately restrained approach to a web presence and later CMS foundation: no unnecessary framework layers, but a clean structure built on tools that remain transparent.

Part of the appeal is exactly that question: how far can you get with well-understood, simple technology before extra abstraction actually becomes useful?

Linux, security and infrastructure work

systems practical work

Not every meaningful project is a single application. A substantial part of my work sits in Linux, administration, hardening, infrastructure, automation and the question of how systems can stay stable and understandable.

That includes scripts, configurations, test environments, security considerations and the sort of technical groundwork that often matters more than the visible surface later on.

Time tracking and access control with Raspberry Pi / RFID

hardware-adjacent concept + implementation

A project at the intersection of hardware, software and practical operations: time tracking and potentially access control using Raspberry Pi, RFID, display hardware and custom logic.

Projects like this are particularly interesting to me because they connect several layers at once — electronics, device setup, application logic, data handling and real-world use.

What this selection is meant to show

The point is not to present every idea as a polished product. What matters more is the range: services, infrastructure, system design, reusable building blocks and projects that grew out of practical needs rather than trend-driven language.

What may still be added later

This page will probably grow over time. Some items deserve detailed project pages later on, while others may remain sketches or workshop notes. Both are perfectly fine, as long as they lead to something useful.
