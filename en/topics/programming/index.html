Software & Programming

Programming is rarely just about a language. More often it is about understanding a problem, breaking it down and shaping a solution that remains readable and useful later on.

That is why this page is not only about individual languages, but also about tools, structure, maintainability and the way software evolves over time.

C, C++ and lower-level work

C and C++ become especially relevant when working closer to systems, performance, tools or hardware. In those areas, structure and precision matter quickly.

Java and C#

Java and C# are often tied to larger applications and environments where maintainability and structure matter.

PHP and pragmatic web development

PHP remains a very useful tool for many web applications and services. It combines directness, wide availability and the ability to grow into more structured systems when needed.

Whenever maintainability, clarity and pragmatic solutions matter, PHP still feels like a natural working environment to me.

JavaScript and web development

JavaScript now belongs naturally to web development, both in the browser and on the server side. What matters here is not only the language, but the interaction with interfaces, APIs, tools and frameworks.

Earlier tools and longer paths

Programming does not always begin with current tools. Earlier work with calculators, Pascal, assembler, Fortran and other systems often leaves a lasting sense of structure, limitations and possibility.

Structure, architecture & maintainability

Good software is rarely just code that happens to work. It also depends on naming, structure, clarity, debugging, documentation and whether a system can still be understood later on.

That is where refactoring, separation of concerns and architectural overview become useful rather than abstract.

Tools & working style

Everyday work naturally includes editors, IDEs, debuggers, build tools, version control and smaller helpers. Tools such as Visual Studio, Visual Studio Code, Git or shell scripts support a way of working that values repeatability and overview.

Debugging, refactoring and incremental improvement belong to programming just as much as writing code.

Web, backend & services

Many current projects live in the space of web applications, REST services, backend logic and user-facing interfaces. This is where classical software questions meet routing, authentication, state and persistence.

It becomes especially interesting when the result is not only functional, but structurally durable as well.
